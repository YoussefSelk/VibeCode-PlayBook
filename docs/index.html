For Humans

How to use this repo in five moves

If you are new to the playbook, this is the quickest reliable path.

01

Install the playbook

Drop the repo files into a real project using the installer scripts in scripts/.

02

Fill the durable docs

Start with project_brief.md, tech_stack.md, active_context.md, and verification.md.

03

Shape rough asks first

04

Assign by ownership

05

Use security, legal, and reviewer when the feature warrants it.

Why It Works

Small docs, clear routing, real verification

Token-efficient by default

The playbook teaches agents to load only the context needed for the current pass instead of dumping the whole repo into every task.

Good for humans too

The same structure that reduces token waste also makes onboarding calmer and easier for new contributors.

Built for real repos

Install it into an existing project and keep your current stack. The playbook should adapt to the repo, not the other way around.

FAQ

Common onboarding questions

Is this a full app framework?

No. It is a workflow layer you install into an existing repo so your agents and contributors follow a clearer operating model.

Why does it have both human docs and LLM docs?

Because this repo is meant to coordinate both. Humans need a clear mental model, and LLMs benefit from compact, explicit entry points.

Why keep the site static?

Static files are easier to host on GitHub Pages, easier to inspect, and simpler for AI agents and developers to parse and maintain.
