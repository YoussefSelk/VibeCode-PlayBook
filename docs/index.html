GitHub Pages guide for this repo

Teach the repo once. Let humans and LLMs onboard fast.

This playbook gives teams a lightweight multi-agent workflow with durable context, clear ownership, and a real verify-before-done habit.

Start Here Open llms.txt

Core idea

  • Keep context short and durable
  • Route work to the right specialist
  • Verify the real runtime flow
  • Finish with a reviewer pass

Best for teams using Codex-style agents without wanting a heavy framework.

For Humans

How to use this repo in five moves

01

Install the playbook

Drop the repo files into a real project using the installer scripts in scripts/.

02

Fill the durable docs

Start with project_brief.md, tech_stack.md, active_context.md, and verification.md.

03

Shape rough asks first

Use prompt-engineer before implementation when the task is vague or crosses layers.

04

Assign by ownership

Let db, backend, or frontend own the change, then validate it with tester.

05

Close with risk review

Use security, legal, and reviewer when the feature warrants it.

For LLMs

The shortest reliable mental model

Load order

  1. Read AGENTS.md
  2. Read agent_docs/active_context.md
  3. Read only the one or two specific docs needed next

Default sequence

prompt-engineer -> owner -> tester -> reviewer

Add security for auth, secrets, trust boundaries, uploads, or sensitive data. Add legal for privacy, consent, retention, disclosures, or risky claims.

Copy-paste prompt

Good first instruction

Have prompt-engineer turn this into a clear execution prompt:
"[rough task here]"

Repo Map

What lives where

AGENTS.md

The operating system for the workflow and ownership rules.

agent_docs/

Small durable context files that reduce repeated prompting.

scripts/

Installers for dropping the playbook into real projects.

docs/

This site and the machine-readable onboarding files for LLMs.

Roles

Who does what

prompt-engineerTurns rough asks into execution prompts.
dbOwns schema, migrations, queries, and persistence contracts.
backendOwns APIs, business logic, validation, auth, and integrations.
frontendOwns UI, accessibility, client behavior, and API wiring.
testerVerifies the real runtime flow across layers.
securityReviews auth, secrets, permissions, trust boundaries, and data exposure.
legalReviews privacy, retention, disclosures, claims, and policy-sensitive flows.
reviewerPerforms final regression and risk review.
